Design a control panel for Missile Guidance Simulation project.
For Google Stitch, ignore the detailed aspect, focus on panel, screen and design. For Claude, focus on the functions, and follow the design outputed by Stitch.
The entire panel should include dark and light mode, while map not affected. Design is modern, clean, minimalist and widget-based.
The web control terminal consists of three screens. Can be navigated from left navigational panel (which can be hidden too). 
·       Planning screen
o   Shall have main dem and map in the middile, and user can select DEM, missile profile, and create missile. On right side, which is the main place where user can set config for the simulation (like start and end gps location). We can also adjust the heurisitc and parameters for pathfinding algorithm. There shall be separation between section of mission config and the pathfinding algorithm. And a display on right side to see current missile configuration. And at bottom right will have a section showing the dem’s config. All those can be move and adjusted and act like widget, so their size can be adjusted, and moved around, can be full screen…
o   At bottom sit the terminal. Which can be implemented to see the progress of pathfinding algorithm.
·       Mission screen
o   This is the most heavy part of the web terminal. In middile, same, we have a big map.
o   Map: Use some third-party maps (which doesn’t have too much label, or at least can be customed) and allows the dem to show its range (in rectangle) on the map, and we are able to add marks and show start/target gps on the map. 
o   The map should include pathfinder’s trajectory, and show missile and show its missile id as label, which depends on missiles’ gps location, it can move on the map. And the map should show missile’s actual trajectory, and pathfinder trajectory but in different colour.
o   On right side, first section on top we shows details of flight, like Navigation: KF result (though tick is 500Hz but update each second only), GPS/TERCOM/INS position and each parameter, and Kalman Filter parameters. And that parameter/monitor section, beside navigation, can also see controls, see controls parameter, pid output, autopilot and flight computer, and also weather/wind condition at the point. Navigated through navigation bar at top of the section. And bottom or  on the right of the map (so basically at middile can be the map + the animation panel), we have a screen which shows tomahawk’s model (simple lines, black/white theme based on dark/light mode) that demonstrated using simple lines, terrain around it based on the dem, draw using lines too, that we can freely turn around at any angle to view 3d model of tomahawk missile and we can see the landscape around it. Again, no beautiful scenery or models. This animation screen should show Tomahawk’s posture, propeller (if attached), trajectory from pathfinding (so we can vislualise how deviated we are) Terminal and Boost guidance route, and the end impact, the pitch will be according to impact angle.
o   Bottom right can show statistics too, which focuses on error and deviation of navigation and guidance. 
o   Lastly, bottom shows like mimic 787’s PFD, although at current moment we don’t have roll pitch, but still shows the PFD, which on left is ground speed, right is altitude (baro or agl, can switch). This is also integrated with main performance monitor, shows missile’s speed, position, engine thrust, and so on.
o   Add a banner which highlight enough at an obvious location showing missile’s flight stage (according to State). And also add time and progress (based on dist to target)
o	Same as before, widget-design, so each widget can be resized, hide and full screen and probably move position (if feasible).
·       Final Report
o   Two sections:
1.     We can view statistics and the entire aeroplane’s flight animation, map position and basically everything in Mission screen, but we can use a scrollbar (horizontal) that act as a timeline, you scroll, drag and show missile’s state and data and posture and visualisation and map at each moment.
2.     Final report, shows the deviation of impact, impact angle, detonated or not, and every other things you find it suitable to do as a final report. And add a way to analyse how successful and this mission is. It can be exported in some format.
